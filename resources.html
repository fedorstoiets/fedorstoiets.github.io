# Fedor Stoietskyi Website A GitHub Pages-ready static website for therapy-informed coaching, life transition support, educational resources, and a biography page. ## Pages included - `index.html` — homepage - `services.html` — services page - `resources.html` — resources/blog hub - `about.html` — biography page - `contact.html` — contact/intro call page - `blog-life-transition.html` — starter blog post - `blog-immigration.html` — starter blog post - `blog-work-meaning.html` — starter blog post ## Before publishing Search the files for these placeholders and update them: - `YOUR-EMAIL@example.com` - `YOUR-GITHUB-USERNAME` - Exact license/supervision/credential language - Real booking link, if using Calendly/TidyCal/etc. - Real testimonials only if you have permission to use them - Replace image suggestion blocks with actual headshots or client-focused stock images ## GitHub Pages deployment ### Option A: User site 1. Create a public GitHub repository named exactly: `YOUR-GITHUB-USERNAME.github.io` 2. Upload all files from this folder into the repository root. 3. Go to **Settings → Pages**. 4. Under **Build and deployment**, choose **Deploy from a branch**. 5. Select `main` and `/root`, then save. 6. Visit: `https://YOUR-GITHUB-USERNAME.github.io` ### Option B: Project site 1. Create any public repository, for example: `fedor-site` 2. Upload all files into the repository root. 3. Go to **Settings → Pages**. 4. Choose **Deploy from a branch**. 5. Select `main` and `/root`. 6. Visit: `https://YOUR-GITHUB-USERNAME.github.io/fedor-site/` ## Contact form note This website uses a static contact form that opens the visitor's email app. For a production practice site, consider using a privacy-conscious contact workflow. If you handle protected health information, use a HIPAA-appropriate workflow. ## Local preview Open `index.html` in your browser, or run: ```bash python -m http.server 8000 ``` Then visit: ```text http://localhost:8000 ```
